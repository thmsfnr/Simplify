Il faut voir comment gérer le paiement, est-ce que mettre en place dans utils quelque chose qui vérifie si le moyen de paiement est fait ou pas.

Je n'ai pas mis les methodes dans les controllers, je te laisse les mettre en fonction de comment tu décides d'implémenter ca.

Pour le flux de gestion responsable, ce n'est pas ici qu'il faut le gérer. Mais côté responsable quand on récupére les réservations, voir si il existe un paiement affecté à cela.


PaiementFrame est la page sur laquelle il faut que le client accede pour renseigner ses infos de paiements
PaiementResume est la page après que le paiement a été accepté
PaiementList est la liste des paiements de l'utilisateur ou la liste des paiements existants que l'admin peut gérer
PaiementDetails est la page sur laquelle on peut consulter toutes les informations de paiements, comme par exemple la commande, le client, les donnees bancaires et les montants;

Faut voir si il faudrait pas faire une seule frame pour PaiementResume et PaiementDetails
